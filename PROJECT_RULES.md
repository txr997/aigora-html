
## Project Standards

You are helping build a production-quality HTML template.

### Code Rules

* Follow existing project file structure.
* Follow existing class naming convention.
* Follow existing SCSS architecture.
* Never create inline CSS.
* Never create inline JavaScript.
* Reuse existing utility classes when possible.
* Keep code ThemeForest quality.
* Write semantic HTML5.
* Use BEM-style naming if existing project uses it.
* Mobile-first responsive structure.
* Keep HTML clean and readable.
* Do not modify unrelated files.
* Check existing sections before generating code.
* Match spacing, typography and layout exactly from Figma.
* Use SCSS nesting style consistent with project.

### Output Rules

* Generate only requested code.
* Do not rewrite unrelated sections.
* Do not rename existing classes unless requested.
* Keep output production ready.
* Prioritize maintainability and performance.

### ThemeForest Rules

* Semantic markup only.
* Accessible HTML structure.
* Responsive implementation.
* No hardcoded content unless required.
* Reusable components preferred.
* Clean indentation and formatting.
